一 PL/SQL块+异常+过程

1 PL/SQL定义

  • PL/SQL是oracle在标准的sql语言上的扩展的一种对oracle数据库进行编程的语言， pl/sql不仅允许嵌入sql语句，还可以定义变量常量，还可以使用条件语句和循环语句
  • 用sql语句编写的缺陷：
    1 不能模块化编程，为了完成下订单，可能我们要发几条sql
    2 执行速度慢，dbms需要重新编译sql语句
    3 安全性问题
    4 比较浪费带款
    —–解决—–> pl/sql编程
  • 简单分类
    块编程：① 过程(存储过程)②函数③触发器④包

    2 存储过程简单demo

  • create procedure pd is begin insert into a(id ,name) values(33,'王大力'); end; drop procedure pd;
  • 调用存储过程
  1. java程序调用
  2. 控制台调用
    exec procedure pd
  • 参数
    oracle过程，可以指定参数是输入参数，还是输出参数，基本语法如下:
    create procedure 过程名(变量名 in 变量类型 ... 变量名 out 变量类型，...) is // 定义变量 begin //执行的语句 end;

    3 pl/sql编写规范

  • 注释
    单行注释：–
    多行注释：/ …. /
  • 标识符号的命名规范
  1. 当定义变量时，建议用v_作为前缀，如v_val;
  2. 当定义常量时，建议用c_作为前缀，如c_rate;
  3. 当定义游标时，建议用_cursor作为后缀，如emp_cursor;
  4. 当定义例外时，建议用e_作为前缀e_error;

    5 参照变量

  • 游标变量 - ref cursor
    通过游标，可以取得返回结果集（往往指向一个select语句返回的结果）的任何一行数据，从而提供共享的效率

PL/SQL试图+触发器+例外+数据库备份恢复

1 视图

  • 视图 是一个虚拟表，其内容由查询定义，同真实的表一样，视图 包含一系列带有名称的列和行数据。但是，视图 并不在数据库中以存储的数据值集形式存在。行和列数据来自由定义视图 的查询所引用的表，并且在引用视图 时动态生成。
  • 主要的用处是：简化操作，提高安全性，满足不同用户的查询需求！
    基本语法：
    create view 视图 名 as select * [with read only]
    如果我们创建视图 的时候，我们带了with read only 表示该视图 只能读，不能进行其他操作！
    eg:
    delete from view_a where id=2;–权限不足
    删除视图
    delete view 视图 名
  • 视图 和表的区别：
    1 表占空间，视图 不需要
    2 视图 不能加索引
    3 视图 可以简化操作

2 触发器

  • 触发器
    for each row;–行级触发器：只要每行修改都会触发触发器！

    –阻止删除员工号为2的员工
    create or replace trigger t1 before delete on a for each row begin if :old.id=153 then Raise_application_error(-20009,'不能删除员工编号为2的用户'); end if; end; commit; select * from a; delete from a where id=153; -- ora-20009 不能删除员工编号为2的用户

  • 监控表的dml操作！
    inserting updating deleting

    create or replace trigger t2
    before insert or update or delete on dacpprod.a
    begin
    case
    when inserting then

    raise_application_error(-20001,'不能新增操作！');

    when updating then

    raise_application_error(-20002,'不能修改操作！');

    when deleting then

    raise_application_error(-20003,'不能删除操作');

    end case;
    end;
    insert into a (id,name) values(3,’a’);– ORA-20001 不能新增操作！
    update a set name=’王二’where id=4; – ORA-20002 不能修改操作！

  • 数据库管理员和对数据库（方案，表）的备份与恢复

    • dba职责
      – 1 安装和升级oracle数据库
      – 2 建库，表空间，表，试图，索引…
      – 3 制定并实施备份与恢复计划
      – 4 数据库权限管理，调优，故障排除
      – 5 对于高级dba,要求能参与项目开发，会编写sql语句，
      – 存储过程，触发器，规则，约束，包。

    • 管理数据库的用户主要是sys和system
      – sys用户存放的数据更重要 system用户对应的方案存放次一级

– 使用import和export对数据库(方案)进行备份和恢复
– import和export逻辑备份与恢复 必须在数据库开启的情况下使用

  • export 导表，导数据库，导方案

    • 导出表
      –基本语法：（该命令需要在dos下执行）
      exp userid=用户名/密码@数据库实例名 tables=(表名1，表名2…) file=备份路径
      – 导出其他方案(用户)的表，则需要dba权限或者是exp_full_database的权限，比如system可以导出scott的表
      exp userid=用户名/密码@数据库实例名 tables=(方案名.表名1，表名2…) file=备份路径
      – 特别说明：备份文件的后缀一般是.dmp但这不是必须的

      exp userid=dacpprod/dacpprod@orcl tables=(dacpprod.a) file=/home/oracle/Desktop/a.dmp

    • 导出表结构
      exp userid=用户名/密码@数据库实例名 tables=(表名1，表名2…) file=备份路径 rows=n
      –使用直接导出方式
      exp userid=用户名/密码@数据库实例名 tables=(表名1，表名2…) file=备份路径 direct=y

    • 2 导出方案 把一个用户整体导出
      – 基本语法：
      exp userid=用户名/密码@数据库实例名 owner=(用户名) file=备份路径
      – 导出其他用户的，则用system方案来owner=(用户名1，用户名2)

    • 导入
      imp userid=用户名/密码@数据库实例名 tables=(a,b) file=备份路径
