oracle学习笔记（一）

发表于 2016-12-04

前言

　　接触oralce一年多了，使用时对oracle的了解并不全面，只是当做数据源去使用，比如包、过程、块、表管理等就不太熟悉，只有一个概念性的了解，作为使用最多的关系型数据库之一，还是很有必要去全面学习了解，因此花了一个月时间对oracle来进行全面地学习，下面就是我系统学习的总结。　　
　　知识点没有前戏(概念性介绍)，知识点间关联也可能不太深，全是干货。

正文

一 oracle管理

1 spool

查询出来显示在屏幕上数据输出到某一个文件
sql> spool /home/oracle/aaa.sql; sql >select * from proc; sql>spool off;
则会在/home/oracle目录下有一个aaa.sql，里面记录了刚才查询的数据

2 设置查询数据每行字符

sql>set linesize 150;

注：sql语句需要带分号（select,update等），其他的可不用带分号(如上面的set linesize等)

3 oracle角色 三种标准角色

Oracle内部有两个建好的用户：SYSTEM和SYS， 用户可直接登录到SYSTEM用户以创建其他用户。

CONNECT Role连接角色

CONNECT　role的用户除了select、insert、update和delete等，还能够创建表、视图、序列（sequence）、簇（cluster）、同义词（synonym ）、会话（session）和与其他数据库的链（link）。

RESOURCE Role(角色)

更可靠和正式的数据库用户可以授予RESOURCE role。RESOURCE提供给用户另外的权限以创建他们自己的表、序列、过程（procedure）、触发器（trigger）、索引（index）和簇（cluster）。

DBA（数据库管理员角色）

DBA role拥有所有的系统权限—-包括无限制的空间限额和给其他用户授予各种权限的能力。SYSTEM由DBA用户拥有。
下面介绍一些DBA经常使用的典型权限:

（1）grant（授权）命令
grant connect, resource to user01;

（2）revoke（撤消）权限
revoke connect, resource from user01;
用法:
create role STUDENT; grant connect,resource to zj_mds;

4 删除用户，必须先删除他的对象

sql>drop user aaa cascade;
解锁用户：
sql>alter user scott account unlock;

5 使用profile文件管理用户口令

比如：

① 只允许某个用户最多尝试登陆三次，如果没成功，就锁定两天

SQL> create profile myprofile limit failed_login_attempts 3 password_lock_time 2; Profile created. SQL> alter user xiaoer profile myprofile ; User altered. SQL> conn xiaoer/x; ERROR: ORA-01017: invalid username/password; logon denied Warning: You are no longer connected to ORACLE. SQL> conn xiaoer/dd ERROR: ORA-01017: invalid username/password; logon denied SQL> conn xiaoer/erer ERROR: ORA-01017: invalid username/password; logon denied SQL> conn xiaoer/eeee ERROR: ORA-28000: the account is locked

② 需求 一个账号最多十天，宽限期2天，到时必须设置新密码

SQL> create profile myprofile1 limit password_life_time 10 password_grace_time 2; Profile created. SQL> alter user daer profile myprofile1; User altered.

③不需要profile可以删除

drop profile myprofile1 SQL> drop profile myprofile1 cascade; Profile dropped.

二 oracle数据类型+表管理

1 文本、二进制类型

主要是varchar1和clob

2 数字类型 number

  • number(p,s)可以存放整数，也可以存放小数，p为有效位，s为小数位
    1<=p<=38;-84<=s<=127
    ex:1 number（5,2）存放-999.99到999.99

  • number（4,2）存123.49 报错

  • number(6,-2) 存 123.49 100

    3 date日期类型

    date默认类型是dd-mm-YYYY(天-月-年);
    如果希望使用自己喜欢的时间格式，则需要借助oracle的函数

timestamp（n）邮戳类型，当数据进行更新时，会自动更新日期

date类型是Oracle常用的日期型变量，他的时间间隔是秒。
timestamp是DATE类型的扩展，可以精确到小数秒（fractional_seconds_precision），可以是0 to9，缺省是６。两个timestamp相减的话，不能直接的得到天数，而是得到， 多少天，多少小时，多少秒等

4 commit

DML操作后需要commit才能生效

表添加字段 alter table numbertest add(age number);
表删除一个字段 alter table numbertest drop column age;
修改表名rename numbertest to numberaaa;
修改表字段的类型 alter table numberaaa modify(price number(20,1));

5 字符串和日期必须加单引号

插入操作，如果给每列都要添加数据，就可以不带列名

6 删除操作

truncate table aaa 一定要小心

delete恢复删除

sql>savepoint aaa; sql>delete from table1; sql>rollback to aaa;
注意：oracle不区分大小写

三 oracle查询

1 NVL(a,0)

如果a 为null,返回0 ，如果不为null，则返回本身

2 || 的使用

在查询的时候，如果希望把多列拼接起来，作为一列返回，可以使用||,具体用法：
select creater || '创建了程序' || proc_name from proc;

3 锁定用户

希望删除用户，同时保留该用户的数据对象，怎么处理?
alter user wanger account lock;
（解锁: alter user wanger account unlock;）

4 between and/like模糊查询/is null 函数

between and 是一个闭区间
like模糊查询:% 表示0到多个字符，_ 表示1个任意字符
is null 函数查询为null的条件

5 order by子句

order by注意是对结果进行排序，默认是升序排列(asc) 降序是desc
oracle 支持使用别名排序
select xmlid 主键id from proc order by 主键id;

6 数据分组的五个函数

max、min、avg、 sum、count
注意：

  • avg 统计时，如果某一个值为null，则不会统计，也不会算平均值(如果要统计null，可用sum(age)/count(*));
  • count(字段)时，也不会去考虑null值

  • 我们书写sql语句是从左到右，但sql默认执行是在从右到左执行。

    7 group by 和having子句(一般两个一起出现)

    group by 用户对查询结果进行分组
    having 用于过滤分组显示结果

  • 1 分组函数(sum.avg)等，只能出现在选择列表、having、orderby中，不能出现在group by中

  • 2 如果select语句中同时包含groupby,having,order bu 那么他们的顺序是group by,having,order by。1
  • 3 在选择列中如果有列、表达式和分组函数，那么这些列和表达式必须有一个出现在group by子句中，否则会出错

8 子查询

  • 一 子查询中使用all
    如何显示工资比部门 30 的所有员工的工资高的员工的姓名、工资和部门号：
    select * from emp where sal >all(select sal from emp where deptno =30)
  • 二 子查询中使用any
    如何显示工资比部门 30 的任意员工的工资高的员工的姓名、工资和部门号：
    select * from emp where sal > any(select sal from emp where deptno = 30)
  • 三 查询与李二的部门和岗位完全相同的雇员
    select * from emp where(deptno,job)=(select deptno,job from emp where ename='李二');

  • 四 查找每个部门工资最高的人的详细信息

    解法1 select t1.* from emp t1,(select max(sal) hsal,deptno from emp group by deptno ) t2 where t2.deptno=t1.deptno and t1.sal=t2.hsal;
    解法2 select * from emp where sal=(select max(sal) from emp e where e.deptno=deptno);
    注意：当在from子句中使用子查询时，该子查询会被作为一个临时表来对待，当在from子句中使用子查询时，必须给子查询指定别名。

    9 分页查询 各个数据库

    mysql:
    select * from 表名 limit 从几条取，取几条；
    oracle:

select t2.* from (select t1.* ,rownum rn from (select * from emp) t1 where rownum<=6) t2 where rn>=4
说明：上面的sql是oracle数据库效率比较的查询方法；在百万级都可以及时响应
oracle使用三层过滤
第一层 select * from emp
第二层 select t1.* ,rownum rn from (select * from emp) t1 where rownum<=6
第三层 select t2.* from (select t1.* ,rownum rn from (select * from emp) t1 where rownum<=6) t2 where rn>=4

实际上我们把上面的sql当做一个分页模板来对待：
6 表示取到第几条
4 从第几条开始取

10 合并查询

  • union
    该操作用于取两个结果集的并集。当使用该操作符时，会自动去掉结果集中重复行
  • unionall
    与union类似，但是不会去重，也不会排序

    11 minus 差集

    用于取得两个结果集的差集，它只会显示存在第一个集合中，而且不存在第二个集合中的数据
    select * fom a.emp minus select * from b.emp;

    12 oracle的内连接

    内连接：是我们用的最多的一种连接，前面我们讲的都是内连接
    ex:比如我们要显示员工的信息和部门名称，select a.*,b.* from table1 a,table2 b where a.No=b.No; == select a.*,b.* from table1 a inner join table2 b on a.No=b.No;

13 外链接

外链接分为：左外连，右连接，完全连接

  1. 左外链接：左侧的表完全显示
    `select a.*,b.* from  table1 a left join table2 b on a.No=b.No;`
    或者 select a.*,b.* from table1 a， table2 b where a.No=b.No(+);

  2. 右外链接：右侧的表完全显示 

    `  select a.*,b.* from  table1 a right join table2 b on a.No=b.No;`

    或者 select a.*,b.* from table1 a ，table2 b where a.No（+）=b.No();

    1. 完全外接：两个表查询，不关有没有匹配上，都列出来
      select a.*,b.* from table1 a full outer join table2 b on a.No=b.No;
