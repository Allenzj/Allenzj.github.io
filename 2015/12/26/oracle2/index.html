oracle学习笔记（二）

前言

　　在oracle学习笔记（一）中我们讲解了有关oracle的简单管理和查询知识点。接下来将会有更多知识点，比如事务，sql函数，PLSQL中的用法等等，接下来都会讲解涉及。

一 sql函数

1 to_date的使用

  • 使用案例
    insert into emp(empno,hiredate) values(222,to_date('1994-12-12','yyyy-mm-dd'));

    2 使用子查询完成更新

  • 使用案例:
    希望员工sc的岗位、工资、补助与smith员工一样
    update emp set(job,sal,comm)=(select job,sal,comm from emp where ename='smith') where ename='sc';

    3 函数分类

  • 单行函数： 对每一行输入值进行计算，得到相应的计算结果，返回给用户，也就是每行作为一个输入参数，经过函数计算得到每行的计算结果，比如length;
  • 多行函数：对多行输入值进行计算，得到多行对应的单个结果，比如max,min

    4 替换函数

  • replace(char,serarch_string,raplace_string);
    将name中姓张的全部替换成李：
    select repalce(name,'张'，‘李’) from emp;

    5 instr(C1,C2,I,J);

  • 在一个字符串中搜索指定的字符，返回出现指定的字符的位置；
    C1 被搜索的字符串，C2希望搜索的字符串，I 搜索开始的位置，默认为1.J 出现的位置，默认为1；

  • ex:请找出‘oracle traning’第二个ra出现的位置
    select instr("oracle traning","ra",1,2) from dual;

6 concat

  • 连接两个字符串
    concat 等同于|| ex:concat(name,’是一个好人’） === name || ‘是一个好人’;

    7 initCap函数

  • 返回字符串并将字符串的第一个字母变为大写；
    SQL> select initcap('penglei') from dual; Penglei

    8 length返回字符串的长度

  • SQL> select length('penglei') from dual;–7

    9 大小写转换

  • Lower
    返回字符串，并将所有的字符小写
  • upper
    返回字符串，并将所有的字符大写

    10 ltrim和rtrim

  • ltrim ()删除左边出现的字符串
  • rtrim()删除右边出现的字符串
    默认为空格
    eg:ltrim("aaayuyua","a");
  • trim（）：去掉指定字符前面或者后面的字符或者数字
    trim("a" from "a232332aa")—–>232332

11 ceil 向上取整

  • 返回大于或者等于给出数字的最小整数
    ceil(3.12)–>4

    12 round函数

  • 用于四舍五入

13 trunc函数

  • 用于截取一个整数
    trunc(44.9) --->44

14 sign

  • 取数字n的符号，大于0返回1，小于0返回-1.等于0返回0

    15 add_months

  • 日期值，增加的月份
    ex:查询三个月内入职的员工
    where add_months(hiredate,3)>=sysdate;

    16 last_day

  • 返回日期的最后一天
    一般是该月或者该年的最后一天！

17 months_between(date1,date2)

  • 计算两个时间相差多少个月
    ex: select months_between('19-12月-1999'，'19-3月-1999') from dual;

    18 to_char（number/date/char）

  • to_char(date,’format’)
  • to_number（varchar2,’format’) 将给出的字符转换为数字
  • to_date（String,’format’）将字符串转换为oracle中的一个日期

    19 系统函数

    sys_context表中的字段
  1. language 语言
  2. db_name 当前数据库名
  3. nls_date_format:当前会话客户所对应的日期格式
  4. session_user:当前会话客户所对应的数据用户名
  5. current_schema:当前会话客户所对应的默认方案名
  6. host:返回数据库所在的主机名称
    eg：select sys_context(‘userenv’,’db_name’)from dual;

二 事务

1 事务的ACID

  • 原子性（Atomicity）
    原子性是指事务是一个不可分割的工作单位，事务中的操作要么都发生，要么都不发生。

  • 一致性（Consistency）
    事务必须使数据库从一个一致性状态变换到另外一个一致性状态。

  • 隔离性（Isolation）
    事务的隔离性是指一个事务的执行不能被其他事务干扰，即一个事务内部的操作及使用的数据对并发的其他事务是隔离的，并发执行的各个事务之间不能互相干扰。

  • 持久性（Durability）
    持久性是指一个事务一旦被提交，它对数据库中数据的改变就是永久性的，接下来的其他操作和数据库故障不应该对其有任何影响。

    2 保存点savepoint

    sql>savepoint aa; sql>delete ... sql>rollback to aa;多个回退点可以多次选择回退，但每个回退点只能选择一次(浪费空间，资源开销大所以只能回退一次)，一旦提交了事务，则不能回退到任何保存点

    3 事务隔离级别

  • 一 脏读
    当一个事务读取另外一个事务尚未提交的修改时，产生脏读。在oracle中没有脏读
  • 二 不可重复读
    同一查询在同一事务中多次进行，由于其他提交事务所做的修改或者删除，每次返回不同的结果集，此时发生非重复读。
  • 三 幻读
    同一查询在同一事务中多次进行，由于其他提交事务所做的插入操作，每次返回不同的结果集，此时发生幻读

4 设置事务隔离级别

SQL标准定义了以下几种事务隔离级别:

READ UNCOMMITTED 幻想读、不可重复读和脏读都允许。

READ COMMITTED 允许幻想读、不可重复读，不允许脏读

REPEATABLE READ 允许幻想读，不允许不可重复读和脏读

SERIALIZABLE 幻想读、不可重复读和脏读都不允许

READONLY 只读事务只能看到事务执行前就已经提交的数据，且事务中不能执行 INSERT ， UPDATE ，及 DELETE 语句

Oracle数据库支持READ COMMITTED、READONLY和 SERIALIZABLE这两种事务隔离级别。所以Oracle不支持脏读

设置事务隔离级别：

已提交读模式：SET TRANSACTION ISOLATION LEVEL＝READ COMMITTED;
串行模式：SET TRANSACTION ISOLATION LEVEL＝ SERIALIZABLE;
只读模式：SET TRANSACTION＝ READ ONLY;

三 序列/索引/管理权限和角色等

1 维护数据的完整性

  • 数据完整性可以使用约束，触发器，应用程序(过程，函数 )
    数据库的完整性用于确保数据库数据遵从一定的商业的逻辑规则。在oracle中，数据完整性可以使用约束、触发器、应用程序（过程、函数）三种方式来实现，在这三种方法中，因为约束易于维护，并且具有最好的性能，所以作为维护数据完整性的首选。

2 约束

  • 一 非空约束（not null）
    用于指定某列的值不可以是null
    usage：
    SQL> create table notNullTable(id int,name varchar2(20) not null); Table created.

  • 二 唯一约束(unique)
    用于指定某列的值不能重复
    usage： SQL> create table uniqueTable(id int,name varchar2(20) unique); Table created.

    notice: unique字段可以为null,也可以有两个为null

  • 三 主键(primary key)
    用于唯一的标示表示的数据，当定义主键约束后，该列不但不能重复，而且不能为null
    primary key 和unique的区别

     1 一个表可以有多个unique,但是只能有一个主键
2 我们每张表都应该有主键，
3 unique列的值可以为null,但主键不能为null
4  primary key的所在列，会自动创建索引，但是unique,不会自动创建索引
  • 四 外键foreign key
    sql: create table b(id int primary key,name varchar2(20),uniqueTableId number references a(id)); Table created.
    注意：a表一定要指定主键,也可以指向unique列
    create table b(id int primary key,name varchar2(20),uniqueTableId number references a(id)); Table created
  • 外检细节说明
    1. 外键指向主键列
    2. 外键可以指向unique
    3. 外键列和主键列的数据类型要一致
    4. 建表时，先建主键，再建从表，删除表的时候，要先删除从表再删除主表。
    5. 外键列的值，必须在主键列中存在
    6. 外键列可以为空null
  • 五 check 检查
    create table c(id int check(id>200)); SQL> insert into c values(20); insert into c values(20); ERROR at line 1: ORA-02290: check constraint (MDS.SYS_C0012454) violated or check (id in (2,3,4,5));

    3 添加约束基本语法说明

  • alter table 表名 add constraint 约束名 约束种类check/unique/primary key/foreign
    alter table 表名 modify 字段名 not null

4 删除约束

语法:
alter table 表名 drop constraint 约束名
约束名：一个表的每一个约束，都对应一个名称

特别说明：因为一张表只能有一个主键，因此在删除主键约束的时候，可以直接使用primary key

alter table c drop constraint check ERROR at line 1: ORA-02250: missing or invalid constraint name 必须指定约束名

5 序列

  • 用来处理自动增长列
    可以为表中的列自动产生值，由用户创建数据库对象，并可由多个用户共享，一般用于主键或唯一列；
    eg:
    create sequence myseq start with 1 increment by 1 minvalue 1 maxvalue 1000000 cycle // nocache; //表示不缓存 ；cache 10表示一次缓存10个号
    用法：
    create table d(id number(10) primary key,name varchar2(20)); Table created. SQL> select * from d; no rows selected SQL> create sequence myswq start with 1 increment by 10 minvalue 1 maxvalue 300 cycle nocache; Sequence created. SQL> insert into d values(myswq.nextval,'abc'); 1 row created. SQL> insert into d values(myswq.nextval,'aaa'); 1 row created. SQL> select * from d;

    ID NAME

11 abc
21 aaa
  • 序列的细则
    1. 可以为表中的列自动产生值
    2. 由用户创建数据库对象，并由多个用户共享。system可以使用scott创建的序列
    3. 一般用于主键或唯一列(unique)
    4. 可以使用序列名.currval来看当前序列到多少号select 序列名.currval from dual
    5. 如果需要去查看序列.currval，必须先使用序列.nextval值，否则出错

      6 索引

  • 如何创建索引
    一 单列索引
    create index 索引名 on 表名(列名）；
    二 多列索引
    create index 索引名 on 表名(列名1，列名2)；
    备注：
    1 在大表上简历索引才有意义
    2 在where子句或者是连接条件上经常引用的列名上建立索引
  • 索引的缺点：
  1. 建立索引，系统要占用大约为表1.2倍的硬盘和内存空间来保存索引
  2. 更新数据时，系统要用额外的时间来更新索引，以维持数据和索引的一致性。
  3. 比如在很少或不用的字段或者逻辑型字段，如男或女等不适用建索引。

    7 oracle权限和角色管理

  • 权限：系统权限和对象权限
    系统权限：对数据库管理的操作以及对数据对象的操作（创建，删除，修改）
对象权限：对数据对象的操作(select,update,insert,delete)
  • with admin option
    表示可以把他得到的权限，继续向别的用户转发
    grant create session to ken with admin optin
  • 对象权限的使用
    指访问其他方案对象的权利
    oracle给我们了17种对象权限，可以通过如下指令查看：
    select distinct privilege from dba_tab_privs;
  • 角色管理
    oracle提供了33种预定义角色，常用的是：connect,dba,resource
    查看某个角色具有哪些权限：
    select * from dba_sys_privs where grantee='DBA'
  • 自定义角色
    建立角色：
    create role 角色名 not identified ;--不设置密码
    create role 角色名 identified by pl --设置验证
  • eg:
    create role myrole not identified; grant create session to myrole;--分配权限 grant select on scott.emp to myrole; --分配权限 grant insert on scott.emp to myrole;--分配权限
    将角色赋予给某个用户pl:
    create user pl identified by 123; grant myrole to pl;
