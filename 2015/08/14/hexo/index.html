使用Github pages基于hexo建个人博客

发表于 2015-08-14

前言

　　工作几年，看很多同事都有自己的博客，于是决定开始写博客，本来打算在CSDN或博客园上写，但觉得广告太多了，影响体验。决定自己搭博客，刚开始想自建网站，无奈太麻烦了，无意中在知乎上发现很多人利用Github pages写博客，于是决定利用github
来建个人博客，使用hexo下的next主题，以下是具体操作以及操作中所踩的坑。

具体操作

软件安装

　　主要是以下软件：
　　 1 nodejs
　　 2 git
　　 3 hexo

安装Node.js

　　为什么要搭建 Node.js 环境？ - 因为 Hexo 博客系统是基于 Node.js 编写的
　　在 Node.js 官网：https://nodejs.org/en/ 下载安装包 v6.10.3 LTS,保持默认设置即可，一路Next。
　　安装完后打开命令提示符，输入 node -v、npm -v，出现版本号则说明 Node.js 环境配置成功！

git安装与github操作

　　下载地址：http://git-scm.com/download/
　　安装后鼠标右击会出现Git Bash 和Git Gui表明安装成功
　　 github注册使用教程：http://www.runoob.com/w3cnote/git-guide.html，这里不是重点，所以略过。
　　
　　 创建仓库(用于存放你的博客页面)：使用自己的用户名，仓库名规则：yourname/yourname.github.io ;
　　 例如我的仓库：allenzj.github.io
　　

hexo安装

Hexo 是一个快速、简洁且高效的博客框架，使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。 建议阅读Hexo的官方文档：https://hexo.io/zh-cn/`
`
windows下右击打开git Bash,执行以下命令：
安装hexo:

$ cd d:/blog #进入目录

$ npm install hexo-cli -g #安装hexo
$ hexo version #查看安装是否完成
$ hexo init allenzj.github.io #初始化hexo allenzj.github.io为文件夹名，你需要与自己之前创建的仓库名保持一致
$ cd allenzj.github.io
$ npm install
$ hexo g # 或者hexo generate
$ hexo s # 或者hexo server，可以在http://localhost:4000/ 查看
此时你就可以在你的浏览器访问http://localhost:4000/查看hexo

注意1：如果不能访问，可能是4000端口被占用，使用hexo s -p 4001指定端口即可。
注意2：执行hexo s提示找不到该指令。在Hexo3.0后server被单独出来了，需要安装server，安装的命令如下：
npm install hexo -server --save

Hexo主题设置

　　hexo的主题是不是太丑了,这里可以使用github上好看的主题，使用主题，可以将博客打造得更加漂亮，我的主题是next，此博文就不详细讲，官网比我写的详细多了：
http://theme-next.iissnan.com/

关联Hexo与Github pages

为什么使用Github

　　GitHub Pages 本用于介绍托管在GitHub的项目，由于他的空间免费稳定，用来做搭建一个博客再好不过了。
　　每个帐号只能有一个仓库来存放个人主页，而且仓库的名字必须是username/username.github.io，这是特殊的命名约定。你可以通过http://username.github.io 来访问你的个人主页。
　　这里特别提醒一下，需要注意的个人主页的网站内容是在master分支下的。

原理

  1. 之前在github上创建的同名仓库，eg:allenzj.github.io实际上里面存放的是html静态文件，里面的master分支可以通过http:allenzj.github.io来访问。
  2. hexo -g命令默认在产生一个public目录，里面存放的静态文件可以直接访问
  3. 将hexo -g生成的默认主题静态文件，提交到github上去

具体操作，使用hexo deploy发布

　　hexo deploy可以部署到很多平台，如果部署到github，需要在配置文件_config.xml中作如下修改：

代码如下：
deploy: type: git repo: https://github.com/Allenzj/Allenzj.github.io.git branch: master
安装一个部署插件：
$ npm install hexo-deployer-git --save
执行部署，在命令行执行：
删除旧的public文件
hexo clean
生成新的public文件
hexo generate or hexo g
部署：
hexo deploye or hexo d
部署即可完成！
注意：
下次在更新blog时：操作如下：
$ hexo clean
$ hexo d -g

此时，访问你的仓库地址 http://用户名.github.io即可在网页上访问
例如我的就是http://allenzj.github.io 访问
